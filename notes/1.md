# C Notes — K&R

---

**1. Preprocessor**

Preprocessor runs before compiler — compiler never sees `#` directives, only expanded result.

- `#include`: copies header file verbatim, replaces directive.
- `#define`: replaces name with value everywhere. `PI → 3.14159`, compiler never sees `PI`.
- **Macros**: `#define` with arguments — pure text substitution, not a function call. No stack/overhead.
  - `#define SQUARE(x) ((x)*(x))` — parentheses critical. `SQUARE(2+3)` without → `2+3 * 2+3`.
- `#ifdef`: includes/excludes blocks at compile time.

> **[bare-metal]** wrap UART debug prints in `#ifdef DEBUG` — release build removes them entirely. Zero overhead, zero flash cost. MCU flash is kilobytes, not gigabytes.

---

**2. Integer Division & Float Promotion**

Integer division truncates. `5/9 = 0`. C standard guarantees this — not hardware-dependent.
At least one float → operation promotes to float. `5.0/9` does not truncate.

- C99: always truncates toward zero.
- C89: negative operands implementation-defined (toward zero or −∞).

> **[bare-metal]** know your compiler.

---

**3. printf Format Specifiers**

`%6.3f` — `6` minimum width, `.3` precision (digits after decimal). Both optional. Look up when needed.

---

**4. Symbolic Constants**

Not variables — no declaration needed.
`UPPER_CASE` for constants, `lower_case` for variables — convention only, not enforced by compiler.
No semicolon at end of `#define`.

---

**5. `main()` — IMP**

`void main()` — invalid in all C standards. `main()` must return `int`.

```c
int main(void) {
    return 0;
}
```

`(void)` ≠ `()` — no arguments vs unspecified arguments. K&R predates this distinction.

> **[bare-metal]** `main()` never returns — infinite loop running the device. Return type still `int`; return statement just never reached.

---

**6. Streams, Buffers, getchar(), putchar()**

- **stream** — abstract flow of characters (concept).
- **buffer** — memory chunk holding stream data temporarily (mechanism).

`getchar()` reads one character at a time from stdin buffer, filled all at once on Enter.
Buffer empty → `getchar()` blocks program until user types more.
`putchar()` prints one character to stdout.

---

**7. Character Constants**

Single quotes → numerical value. `'A'=65`, `'Z'=90`, `'a'=97`, `'\n'=10` (ASCII).
Comparing `int` to `'A'` is valid C — `'A'` is just `65`.
`getchar()` returns `int`, not `char`.

---

**8. stdin, stdout, stderr**

| Stream | Default |
|--------|---------|
| `stdin` | keyboard |
| `stdout` | terminal screen |
| `stderr` | terminal screen (separate stream, for errors) |

All three exist at program startup automatically. Can be redirected via shell.

> **[bare-metal]** no `stdin`/`stdout`. UART replaces them — sends text from MCU to laptop terminal. Effectively `printf()` for microcontrollers.

---

**9. Evaluation Direction**

- **Comma operator** `a, b` — left to right. Two expressions where one expected. Avoid writing; recognise in others' code.
- `&&` and `||` — left to right; stops as soon as result known. If `a` is false in `a && b`, `b` never evaluated.
- **Chained assignment** — right to left. `nl = nw = nc = 0` → `nl = (nw = (nc = 0))`.

> **[bare-metal]** put cheaper expression first in `&&`/`||` — if false, expensive one never runs. saves cycles in tight loops.

Assignment is an expression with a value, not a special case —
`nc = 0` sets `nc` to zero AND returns `0`. that return value is what `nw` receives.
This is also why `while ((c = getchar()) != EOF)` works — `c = getchar()` assigns to `c`
AND returns the assigned value, which is then compared to `EOF`.
